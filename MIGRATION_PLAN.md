# خطة الانتقال إلى السيرفر الخاص (VPS Migration Plan) 🚀

هذا الدليل مخصص للانتقال من Vercel إلى سيرفر خاص (Hetzner/DigitalOcean) باستخدام **Coolify**.

## 1. التجهيزات المطلوبة (Infrastructure)
* [ ] **حساب في مزود سيرفرات:** ننصح بـ [Hetzner](https://www.hetzner.com/) (الأفضل سعراً وأداءً) أو DigitalOcean.
* [ ] **إنشاء سيرفر جديد (VPS):**
    * النوع: Shared CPU (مثلاً CX22 في Hetzner).
    * النظام: Ubuntu 22.04 LTS.
    * الموقع: اختر أقرب منطقة جغرافية لجمهورك.

## 2. تثبيت لوحة التحكم (Coolify)
بمجرد الدخول للسيرفر عبر الـ Terminal، قم بتشغيل هذا الأمر الواحد فقط:
```bash
curl -fsSL https://cdn.coolify.io/install.sh | bash
```
هذا الأمر سيقوم بتثبيت كل ما تحتاجه (Docker, Coolify, Reverse Proxy) تلقائياً.

## 3. إعداد المشروع في Coolify
1. افتح رابط لوحة التحكم (سيكون على الـ IP الخاص بسيرفرك على المنفذ 8000).
2. اختر **Sources** واربط حسابك في GitHub.
3. اختر **Resources** ثم **New Application**.
4. اختر المستودع (quran-platform) والفرع (dev).
5. سيتعرف Coolify تلقائياً على ملف `Dockerfile` الذي أنشأناه.

## 4. إعدادات البيئة (Environment Variables)
* [ ] قم بنسخ كافة المتغيرات الموجودة في ملف `.env` ووضعها في قسم **Environment Variables** في Coolify.
* [ ] تأكد من صحة رابط قاعدة البيانات (DATABASE_URL) ورابط Firebase.

## 5. قاعدة البيانات (Database)
لديك خياران:
1. **الاستمرار مع الخارجي:** ابقِ على Supabase/Neon كما هو (فقط حدث الرابط في الإعدادات).
2. **الاستضافة الذاتية:** يمكنك تشغيل PostgreSQL داخل نفس السيرفر عبر Coolify (يوفر لك المال والسرعة).

## 6. المرحلة النهائية (DNS Switch)
بعد التأكد من أن الموقع يعمل بشكل مثالي على الرابط المؤقت الذي يوفره Coolify:
1. اذهب للوحة تحكم الدومين (GoDaddy / Cloudflare).
2. قم بتغيير سجل الـ **A Record** ليوجه إلى الـ IP الجديد للسيرفر.
3. انتظر دقائق.. ومبارك عليك الاستقلال التام! 🎊

---
**ملاحظة:** ملفات `Dockerfile` و `docker-compose.yml` موجودة الآن في جذع المشروع وهي جاهزة للعمل فوراً.
