# دليل الأمان والجودة - منصة معالي 🛡️

هذا الدليل يهدف لضمان استقرار وأمان المنصة بعد تحولها لبيئة عمل فعلية (Production).

## 1. دورة التطوير الآمنة
1. **التطوير المحلي:** يتم تنفيذ الكود وتجربته على الجهاز الشخصي أولاً.
2. **الاختبار اليدوي:** تجربة الميزة الجديدة بثلاثة حسابات مختلفة (مشرف، معلم، طالب) للتأكد من صلاحيات الوصول.
3. **فحص الثغرات:** التأكد من عدم وجود بيانات حساسة (Passwords, API Keys) مكتوبة بشكل صريح في الكود.
4. **الرفع (Push):** يتم الرفع للمستودع بعد التأكد من خلو الكود من أي "Console Logs" أو بيانات اختبار.

## 2. قائمة الأمان (Security Checklist)
- [ ] **Middleware:** هل الصفحة محمية وتفحص نوع المستخدم؟
- [ ] **API Security:** هل الـ Route يتأكد من هوية الشخص الذي يطلب البيانات؟
- [ ] **Data Sanitization:** هل نقوم بتنظيف المدخلات لمنع الـ SQL Injection؟
- [ ] **Environment Variables:** هل جميع الروابط السرية موجودة في ملف `.env`؟

## 3. خطة الطوارئ (Disaster Recovery)
- **النسخ الاحتياطي:** يتم أخذ نسخة من قاعدة البيانات كل 24 ساعة.
- **التراجع (Rollback):** في حال حدوث خطأ كارثي بعد التحديث، نستخدم الأمر التالي للعودة لآخر نسخة مستقرة:
  ```bash
  git log # لمعرفة الـ ID الخاص بآخر نسخة مستقرة
  git reset --hard <COMMIT_ID>
  git push -f origin main
  ```

## 4. نصائح للمستقبل
* **التحديثات الدورية:** تحديث المكتبات البرمجية (npm packages) لترقيع الثغرات الأمنية المكتشفة.
* **كلمات المرور:** فرض تغيير كلمات المرور الضعيفة للمشرفين والمعلمين.
